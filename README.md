# SVG Font Glyph Width Estimator
Uses javascript and a browser to output text metrics;

To use this, clone the repository and open index.html in your browser. 
main.js does some invisible measuring as the page loads and outputs the result to the page.
The output is JSON (sortof) with width and height for each glyph.
You can copy this output directly into javascript or python projects.

**Note** that outputs are divided by the font-size and are therefore unitless.
When estimating text width you need to multiply the sum of the glyph widths by your font-size.

For really big or really small font sizes you may wish to tweek main.js to render the fonts in a different size before it measures the glyphs.
Currently all measurements are done on font-size=15 (which corresponds to css font-size=15px).
I am not sure, but maybe some fonts have different proportions at different font-sizes?

This script would be much improved by sampling kerning pairs, fork away!

## sample implementation
If you paste the results below into some python module (i suggest **font_profiles.py**) then the following snippet will estimate the width of a given string</p>

```python
from . import font_profiles

def width_estimate(line_of_text: str, font_size: float) -> float:
	linewidth = 0
	for char in line_of_text:
		if char in font_profiles.arial:
			linewidth += font_profiles.arial[char]["width"]
		else:
			linewidth += font_profiles.arial["_"]
	return linewidth * font_size

print(f"Width of 'lorem ipsum' in arial size 12px = {width_estimate('lorem ipsum', 12)}")
```
